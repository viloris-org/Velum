# Security Policy

## Supported Versions

Velum has no supported release and makes no production security claim. The
current documents and code are research artifacts; protocol decisions remain
reversible until their roadmap evidence gates close.

## Reporting a Vulnerability

After this repository is hosted on GitHub, report vulnerabilities through
GitHub Private Vulnerability Reporting. Do not include exploit details,
credentials, packet captures, or other sensitive material in a public issue.

Until Private Vulnerability Reporting is enabled, there is no approved private
reporting channel. That setup is a Foundation exit requirement and must not be
represented as complete based on this file alone.

Reports will be assessed by the security-maintainers responsibility group.
No response or remediation SLA exists before a supported release is declared.
